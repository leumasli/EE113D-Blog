Week 4: Solving Linear Equations with HHL algorithm

This week, I started studying quantum subroutines in machine learning. One common task is to solve linear equations. Harrow et al. introduced an algorithm in the paper Quantum Algorithm for Linear Systems of Equations in 2009 an algorithms (HHL algorithm) solving x→ in the system Ax→=b→ given A and b→ which provided exponential speedup over classical algorithms under ideal conditions.

  • Given a matrix A and a vector b→, find a vector x→ such that Ax→=b→
  • If the matrix is s-sparse, the best classical algorithm has complexity O(Nsκlog⁡(1/ϵ)) where κ denotes the condition number of the system and ϵ the accuracy of the approximation.
  • This paper introduced the HHL algorithm which is a quantum algorithm has complexity O(log⁡(N)s2κ2/ϵ) which provides up to exponential speedup over classical algorithms

Problem Setup

  • Encode the equation in the quantum language, we get
A|x⟩=|b⟩
  • Because A=A†, we can write A in terms of its spectral decomposition
A=∑j=0N−1λj|uj⟩⟨uj|,λj∈R
  • Apply λ−1A on both sides of the eigenvalue equation, we get
A−1=∑j=0N−1λj−1|uj⟩⟨uj|
  • We can also express |b⟩ in the eigenbasis of A
|b⟩=∑j=0N−1bj|uj⟩

HHL Algorithm

  • Load |0⟩nb↦|b⟩nb
  • Apply QPE with
U=eiAt:=∑j=0N−1eiλjt|uj⟩⟨uj|

The quantum state after QPE is

∑j=0N−1bj|λj⟩nl|uj⟩nb
  • We now add an auxiliary qubit and apply a conditional rotation with control qubit as nl
∑j=0N−1bj|λj⟩nl|uj⟩nb(1−C2λj2|0⟩+Cλj|1⟩)
  • Apply inverse QPE which resets |λj⟩nl to |0⟩
  • Measure the auxiliary qubit. If the outcome is |1⟩, the register is in the state
(1∑j=0N−1|bj|2/|λj|2)∑j=0N−1bjλj|0⟩nl|uj⟩nb

Which corresponds to the solution up to a normalization factor.

Notes

  • If A is not Hermitian, construct
C=(0AA†0)

and solve the equation Cy→=(b→0) to obtain y=(0x→)

  • Note that HHL with O(log⁡(N)s2κ2/ϵ) is more sensitive to sparsity, condition of the matrix and error than traditional algorithms with O(Nsκlog⁡(1/ϵ))
  • The complexity of HHL is roughly O(log⁡(N)) which is actually less than printing out all the terms of the result state |x⟩ which is O(N)! In realistic situations, we usually don't care about the specific values of x but rather some other expressions such as the expectation value of some operator ⟨x|M|x⟩. HHL can be a useful subroutine for such tasks.
